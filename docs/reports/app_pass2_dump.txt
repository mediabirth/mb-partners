■ ④ パートナー登録フロー（4ステップ＋完了）— 添付HTML準拠
  画面: app/invite/[token]/InviteForm.tsx を4ステップ・ウィザードに再構築（プログレスバー＋ステップ遷移）。
   STEP1 アカウント: メール(招待値・読取専用)＋パスワード＋確認（認証は現行パスワード方式を維持）
   STEP2 基本情報: 姓*・名*・ニックネーム(任意)・電話*・住所*
   STEP3 報酬受取: 税区分*(個人/法人)・銀行*・支店*・種別*(普通/当座)・口座番号*・口座名義カナ*・インボイス(任意)
   STEP4 確認と同意: 入力サマリ＋パートナー規約/プライバシー同意（両チェックで「登録を完了する」活性）
   完了: パートナーコード表示→「ダッシュボードへ」。
  保存: POST /api/invite/accept を拡張。
   - profiles: name(姓 名), nickname を保存/更新。
   - partners: tax_type, bank(jsonb: bank_name/branch_name/account_type/account_number/account_holder),
     phone, address, invoice_number, terms_agreed_at, privacy_agreed_at（同意は取得日時）を保存。
   - 既存の招待発行/メール送信(Resend)・使用済みマークは維持。完了時に code を返却。
  ★スキーマ追加が必要（本番未作成→勝彦が SQL Editor で実行・IF NOT EXISTS・加算的）:
     supabase/migrations/20260619000001_partner_registration_fields.sql
       alter table partners add column if not exists phone text;
       alter table partners add column if not exists address text;
       alter table partners add column if not exists invoice_number text;
       alter table partners add column if not exists terms_agreed_at timestamptz;
       alter table partners add column if not exists privacy_agreed_at timestamptz;
     ※未作成だと STEP4 送信時に partners 更新が失敗。デプロイ前に作成必須。
  検証(ローカル・テスト招待): STEP1→2→3→4 を遷移、確認サマリ表示、同意未チェックで完了ボタン不活性を確認。
