# M1 完了レポート

## 概要

M1（認証・権限・ルートガード）の実装を完了し、Playwright スモークテスト 6 件全件 PASS を確認しました。

---

## 実装内容

### 認証フロー

| ルート | 対象 | 認証方式 |
|---|---|---|
| `/login` | パートナー | マジックリンク（メール送信） |
| `/console/login` | 管理者 | メール + パスワード + TOTP 2段階認証 |
| `/auth/callback` | 共通 | 認証コールバック → ロール判定 → リダイレクト |

### ルートガード（`proxy.ts`）

未認証・ロール不一致・AAL2 未達をすべて middleware レベルでブロックし、適切なログインページにリダイレクト。

| チェック項目 | 処理 |
|---|---|
| 未認証 → `/app/*` | `/login` にリダイレクト |
| 未認証 → `/console/*` | `/console/login` にリダイレクト |
| `partner` ロール → `/console/*` | `/app` にリダイレクト |
| `owner` ロール → `/app/*` | `/console` にリダイレクト |
| TOTP 未設定の管理者 | TOTP 登録フローへ誘導 |
| AAL1 の管理者 → `/console/*` | TOTP 入力フローへ誘導 |

### ページ・コンポーネント

| パス | 説明 |
|---|---|
| `/app` | パートナーホーム（骨格） |
| `/console` | 管理コンソールホーム（骨格） |
| `components/LogoutButton.tsx` | ログアウト共通コンポーネント |

### デザイン基盤

- グローバル CSS にデザイントークン（カラー・スペーシング）定義
- フォント: Inter（英数）+ Zen Kaku Gothic New（日本語）

## 技術メモ

- Next.js 16 では `middleware.ts` が廃止され `proxy.ts` が正式ファイル名
- Supabase TOTP: `supabase.auth.mfa.enroll()` → QR 表示 → `supabase.auth.mfa.verify()` → AAL2 昇格
- `profiles` テーブルの `role` カラムは `'owner'` / `'partner'`（`'admin'` は無効）
