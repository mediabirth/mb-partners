■ ①【perf】APP報酬の異常な遅さ（8秒）
  原因特定（prod実測）: /app/rewards の初回 = 11,338ms（cold start）、warm ~650ms。
   → Node サーバーレスの cold start が主因（重いクエリ/N+1ではない。rewardsは getPartnerWithDeals の
     単一埋め込みクエリ。payouts API は既に edge＋単一クエリ）。
  対応:
   - サーバーComponent の APP ページ7本を edge 化（cold start ほぼ0）:
     app/page, cases, cases/[id], rewards, rewards/statement, mypage, guide に
     `export const runtime = 'edge'` 追加（dashboard と同じ edge 互換パターン）。
   - 支払管理ページ: 不要だった /api/console/deals（全件取得・重い）フェッチを撤去
     （profile プロップは ConsoleSession 共有で不要に）。payout バッチ取得のみに。
  → prodデプロイ後に rewards TTFB を再計測して before/after 報告。

■ ④ モーション削除（サービス・報酬 と 支払管理 の数字）
  - 支払管理: 合計/源泉/手取 の <CountUp> を撤去し ¥即時表示（CountUp import も削除）。
  - サービス・報酬(ServicesClient): 固定額の <CountUp> を ¥即時表示に（import削除）。
  - ダッシュボード/APP報酬ヒーロー等の高揚モーションは維持（指定どおり対象2ページのみ）。

■ ⑤ 支払サイクル

    （fixed_month 基準）で動作 → 実質「月末締め翌月末払い」固定。変更不可・非表示。
