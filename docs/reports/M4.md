# M4 完了レポート

## 概要

M4（配信・問い合わせ・通知）の実装を完了し、Playwright E2E テスト 15 件全件 PASS を確認しました。

---

## 実装内容

### STEP 1: 通知基盤（`lib/notifications.ts`）

```typescript
createNotification(supabase, partnerId, title, body?, ref?)
createNotifications(supabase, items[])
```

`notifications` テーブルへの書き込みを一元化。全通知種別を `NotificationRef` 型で管理:

| type | 用途 |
|---|---|
| `deal` | 新規案件受付 |
| `payout` / `payout_paid` | 報酬通知 |
| `broadcast` | 配信記事 |
| `inquiry_reply` | 問い合わせ返信 |

### STEP 2A: 配信（Broadcasts）

**DB テーブル:** `broadcasts`, `broadcast_reads`

| ルート | 機能 |
|---|---|
| `GET /api/console/broadcasts` | 配信一覧（開封数付き） |
| `POST /api/console/broadcasts` | 新規作成 |
| `PATCH /api/console/broadcasts/[id]` | 配信送信（通知送信） |
| `DELETE /api/console/broadcasts/[id]` | 未配信のみ削除可 |
| `POST /api/broadcasts/[id]/read` | 開封記録（upsert） |

**UI ページ:** `/console/broadcasts`, `/console/broadcasts/new`, `/console/broadcasts/[id]`, `/console/broadcasts/[id]/preview`

### STEP 2B: 問い合わせ（Inquiries）

**DB テーブル:** `inquiries`, `inquiry_messages`

| ルート | 機能 |
|---|---|
| `GET /api/inquiries` | パートナーの問い合わせ一覧 |
| `POST /api/inquiries` | 問い合わせ新規送信 |
| `GET /api/inquiries/[id]` | スレッド詳細 |
| `POST /api/inquiries/[id]/messages` | パートナー追加返信 |
| `GET /api/console/inquiries` | 管理者: 全問い合わせ一覧 |
| `GET/POST /api/console/inquiries/[id]` | 管理者: スレッド詳細・返信 |

**UI ページ:** `/app/support`, `/app/support/[id]`, `/console/inquiries`, `/console/inquiries/[id]`

### 通知センター（`/app/inbox`）

2 タブ構成:
- **あなた宛**: 案件・支払・問い合わせ返信の通知（未読バッジ・既読マーク）
- **お知らせ**: 配信記事一覧
