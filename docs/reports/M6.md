## M6-2: 口座変更 KYC 承認フロー

**ステータス: 実装・E2E テスト完了（6/6 PASS）**

### 設計方針

- **承認時のみ** `partners.bank` を更新する
- **却下時は絶対に** `partners.bank` を更新しない
- pending 申請が存在する間は二重申請をブロック（409）
- 却下後は再申請可能

### DB スキーマ

既存 `bank_change_requests` テーブルに以下を追加:
- `before_bank JSONB` — 申請時点の口座スナップショット（差分表示用）
- `status CHECK('pending', 'approved', 'rejected')` — 制約追加
- RLS ポリシー（partner: 自分の申請のみ / admin: 全件）

`partners.bank JSONB` の構造:
```json
{
  "bank_name": "銀行名",
  "branch_name": "支店名",
  "account_type": "普通 or 当座",
  "account_number": "口座番号",
  "account_holder": "口座名義（カナ）"
}
```

### 実装ファイル

| ファイル | 内容 |
|---|---|
| `supabase/migrations/20260613000040_m6b_bank_change_requests.sql` | before_bank 追加・status 制約・RLS |
| `app/api/bank-change-requests/route.ts` | POST: 申請作成 / GET: 申請一覧 |
| `app/api/console/bank-change-requests/[id]/route.ts` | PATCH: 承認 or 却下（承認時のみ bank 更新） |
| `app/app/rewards/BankChangeSection.tsx` | パートナー向け口座変更フォーム（client component） |
| `app/app/rewards/page.tsx` | BankChangeSection を追加 |
| `app/console/partners/[id]/page.tsx` | 管理者向けパートナー詳細ページ（銀行変更申請パネル含む） |
| `app/console/partners/[id]/BankChangePanel.tsx` | before/after 差分表示 + 承認/却下 UI（client component） |
| `app/console/partners/page.tsx` | 各パートナーへの詳細ページリンクを追加 |

### E2E テスト結果

|---|---|
| M6B-1: POST /api/bank-change-requests → 201 & id が返る | ✓ PASS |
| M6B-2: pending がある状態で再申請すると 409 | ✓ PASS |
| M6B-3: **承認後に partners.bank = new_bank になる** | ✓ PASS |
