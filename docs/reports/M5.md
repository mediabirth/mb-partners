# M5 完了レポート — Google Calendar 連携

## 実装概要

パートナーが Google カレンダーと連携し、クライアントが空き枠を確認して予約できる機能を実装した。

## 設計ポイント

### セキュリティ
- OAuth トークン（access_token / refresh_token）は AES-256-GCM で暗号化して `calendar_links.oauth_tokens` に保存
- state パラメータ（base64url JSON: `{ partner_id, nonce }`）で CSRF 対策
- `prompt: 'consent'` + `access_type: 'offline'` で refresh_token を毎回取得

### 空き枠算出
- `calcAvailableSlots` が JST (UTC+9) で曜日・時刻判定
- Google FreeBusy API で既存予定を取得し、バッファ付きで重複除外
- access_token 期限切れ時は自動リフレッシュ（更新トークンを DB に保存）

### 予約フロー
- Google Calendar イベント作成失敗時も `meetings` への保存は続行（graceful degradation）
- 予約確定時に `createNotification` でパートナーへ通知

## 手動確認が必要な項目

以下は実際の Google アカウントを使った OAuth 往復のため、本番環境でのみ手動確認可能：
- パートナーが `/app/calendar` → Google と連携する → OAuth 認可 → `calendar_links` にトークン保存
- 連携後に空き枠が `/book/[partner_id]` に正しく表示される
- 予約確定後にパートナーの Google カレンダーにイベントが追加される
