## 提案の骨子（不採用）
- 基調: 紙白（暖色ニュートラル `#F6F4EE` 系）× 墨黒 `#1B1A17`。
- 角丸ほぼゼロ（4px）、999px ピル廃止。
- インディゴ `#4733E6` は色相不変のまま「署名」用途に限定（金額下線数字・リンク・選択・フォーカス・ロゴ）。塗りボタンは墨。
- 署名: RewardPill → RewardFigure（インディゴ下線付き等幅数字）。
- ダーク「墨と絹」を `:root[data-theme]` で定義。

## 差し戻しの学び（記録）
- トークン化率が高いため `:root` 差替で全面が一気に変わる＝**強力だが、実機の総合的な印象は白基調の明るさ/親しみを損なった**。
- ダーク基盤はハードコード `#fff` カードが多く、token 化前は半反転になる（本番未点灯で正解だった）。
- 将来もし再挑戦するなら、(1) 白基調は維持しつつ**署名としてのインディゴ規律**（塗りの乱用抑制）だけを先に部分適用、(2) `#fff`→token 化を先行、の順が安全。
